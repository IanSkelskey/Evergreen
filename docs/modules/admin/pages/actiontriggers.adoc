= Notifications / Action Triggers =
:toc:

== Introduction ==

indexterm:[action triggers, event definitions, notifications]

Action Triggers allow administrators to automate actions based on specific events in the system. They are useful for tasks such as sending notifications, generating reports, or updating records. This functionality helps streamline workflows and ensure timely responses to important events.

To access the Action Triggers module, navigate to *Administration -> Local Administration -> Notifications / Action Triggers*.

[NOTE]
==========
You must have Local Administrator permissions to access the Action Triggers module.
==========

== Key Features ==

The Action Triggers module includes the following components, each accessible through dedicated tabs:

* <<event_definitions, Event Definitions>>: Define the conditions under which triggers are activated.
* <<hooks, Hooks>>: Specify the system events or actions that serve as trigger points.
* <<reactors, Reactors>>: Define the actions to be executed when triggers are activated.
* <<validators, Validators>>: Set the criteria to validate whether triggers should proceed.

Each of these components plays a critical role in configuring and managing automated workflows. The following sections provide an overview of how these components interact and how to navigate the module.

== Quick Start Guide ==

Follow these steps to set up and manage Action Triggers:

. Navigate to the *Notifications / Action Triggers* module.
. Use the *Event Definitions* tab to create or manage triggers.
. Assign a Hook, Reactor, and optional Validator to the trigger.
. Configure additional options such as processing delays, templates, and granularity.
. Save your changes and test the trigger as needed.
